# PokerTimerPLUS+ 機能仕様書 (specs.md)

> 本書は、既存の **PokerStars Poker Clock (Adobe AIR版)** を参考に、全国のポーカールーム向けに無料配布する **Electron 製ポーカークロック PokerTimerPLUS+** を新規開発するための仕様書である。
> 既存アプリのコード・素材は直接コピーせず、機能を言語化したものを記す。
> 不明確な箇所は **(推定)** と明記する。

- **プロダクト名**: PokerTimerPLUS+
- **制作**: Yu Shitamachi（PLUS2運営）
- **配布形態**: 全国のポーカールームへ無料配布

---

## 1. アプリ概要

- **目的**: ポーカートーナメントの進行管理（ブラインドタイマー、参加者数、賞金プール、ペイアウト表示）を、各ポーカールームのディスプレイで運用する。Yu Shitamachi（PLUS2運営）が制作した無料配布アプリ。
- **形態**: Electron 製デスクトップアプリ (Windows / macOS)。完全ローカル動作。
- **想定ディスプレイ**: 全国のポーカールーム・店舗の大型ディスプレイ／プロジェクタ、フルスクリーン表示。複数モニタ対応 (Phase 2 以降の検討事項)。
- **ユーザー**: ディーラー／フロアスタッフ。マウス＋キーボード操作中心、運用中はキーボードショートカット主体。
- **言語**: UI は完全日本語化。プリセット名やショートカット説明含む全文言を日本語で表記。ただしアプリ名 `PokerTimerPLUS+` は固定。

## 2. 機能一覧 (カテゴリ別)

### 2.1 タイマー (Clock)
- レベル別カウントダウンタイマー
- 一時停止 / 再開
- リセット
- レベル手動進行 (次レベル / 前レベル)
- レベル時間の手動加減 (±1 分)
- 経過時間の累積管理 (トーナメント全体の総経過時間)
- ブレイク (休憩レベル) 対応
- スタート前カウントダウン (3-2-1 表示)

### 2.2 ブラインド構造管理
- レベルごとの SB (スモールブラインド) / BB (ビッグブラインド) / Ante 設定
- レベル時間 (分) 設定
- ブレイクレベル (level=null) の挿入
- プリセット保存・読み込み (JSON ファイル)
- レベル追加 / 削除 / 並び替え
- CSV インポート (Phase 2)

### 2.3 プレイヤー管理
- 参加者数の手動加減 (+1 / -1)
- バイイン件数管理 (初期参加者数)
- リバイ件数管理
- アドオン件数管理
- 平均スタックの自動計算

### 2.4 賞金 (ペイアウト) 管理
- 総プール額の自動計算 (バイイン×件数 + リバイ×件数 + アドオン×件数)
- 順位別賞金分配 (パーセンテージ or 固定額)
- 残り人数に応じた ITM (In The Money) 表示
- 賞金額の丸め (1円 / 100円 / 1000円単位)
- ペイアウトプリセット (例: 9人時は3位まで、27人時は5位まで)

### 2.5 通知音
- レベル終了時のチャイム
- 1分前警告音（残り60秒）
- 10秒前警告音（残り10秒）
- **5秒前カウントダウン音（残り5/4/3/2/1秒で各1回、計5回）**
- スタートカウントダウン音（3, 2, 1）
- 音量調整 / 個別 ON/OFF

### 2.6 表示・レイアウト
- フルスクリーン切替
- 店舗ロゴ表示（任意のロゴをユーザーがアップロード可能、初期値は同梱のPLUS2ロゴ）
- 背景画像／背景色のカスタマイズ
- 大型タイマー数字表示
- 現レベル / 次レベル の SB/BB/Ante 同時表示
- ペイアウト表のローテーション表示 (人数が多い場合)
- マーキー（告知スクロールテキスト、最下段に固定配置、ON/OFF・速度3段階・テキスト編集対応）

### 2.7 設定永続化
- ブラインド構造プリセット
- ペイアウトプリセット
- 音量・通知設定
- ロゴ・背景画像パス
- 直前のトーナメント状態 (復帰用)

### 2.8 トーナメント情報
- イベント名 / サブタイトル設定
- バイイン金額 / バイインチップ数
- リバイ金額 / リバイチップ数 / リバイ可能期間
- アドオン金額 / アドオンチップ数 / アドオンタイミング
- 通貨記号 (¥ デフォルト、設定で変更可)

## 3. 各機能の入力・出力・制約

### 3.1 タイマー
| 項目 | 内容 |
|------|------|
| 入力 | スタート/ポーズ/リセット ボタン、ショートカットキー |
| 出力 | MM:SS 形式の大型表示 (残り0秒近くで HH:MM:SS にはしない) |
| 制約 | `performance.now()` 基準のループで±100ms以内の精度を維持 |
| 制約 | 一時停止中はカウントしない、ウィンドウ非アクティブでも継続 |

### 3.2 ブラインド構造
| 項目 | 内容 |
|------|------|
| 入力 | レベル番号、SB、BB、Ante、レベル時間 (分)、break フラグ |
| 出力 | 現レベル + 次レベル の SB/BB/Ante、レベル番号 |
| 制約 | SB ≤ BB、Ante ≥ 0、レベル時間 ≥ 1 分 |
| 制約 | 1 構造あたり最大 50 レベル (推定)、全レベル合計 12 時間まで |

### 3.3 プレイヤー数
| 項目 | 内容 |
|------|------|
| 入力 | +/- ボタン、↑↓ キー |
| 出力 | 残り人数、平均スタック (= 全チップ ÷ 残り人数) |
| 制約 | 残り人数 ≤ 総バイイン数 + リバイ数 + アドオン数 |
| 制約 | 残り人数 ≥ 1 (0 でトーナメント終了表示) |

### 3.4 ペイアウト
| 項目 | 内容 |
|------|------|
| 入力 | 順位ごとの % または固定額、丸め単位 |
| 出力 | 順位別賞金額、合計額 = 総プール |
| 制約 | % 合計は 100.0% ± 0.1% (丸めによる誤差許容) |

### 3.5 通知音
| 項目 | 内容 |
|------|------|
| 入力 | 音源ファイル (mp3)、再生タイミング設定 |
| 出力 | スピーカー出力 |
| 制約 | mp3 のみ対応、ファイルサイズ 5MB まで (推定) |

## 4. 画面構成

### 4.1 メインクロック画面 (運用画面)
最も使用頻度が高い画面。フルスクリーン表示を前提とする。

**配置 (16:9 想定、3列グリッド)**

```
┌──────────────────────────────────────────────────────────────────┐
│ [ロゴ]       イベント名 (大)             NEXT BREAK IN           │
│              サブタイトル / レベル N        HH:MM:SS              │
│                                                                  │
│  1位 ¥75,000           ┌──────────────┐                          │
│  2位 ¥45,000           │              │           AVG STACK      │
│  3位 ¥30,000           │    12:34     │            233,249       │
│                        │  (大型タイマー) │                          │
│                        │              │                          │
│                        └──────────────┘           PLAYERS        │
│                                                    30 / 33       │
│                       BLINDS  500 / 1,000                        │
│                       ANTE    100                                │
│                       NEXT    600 / 1,200 / 200 (やや小)         │
│                                                                  │
│  TOTAL PRIZE POOL                                Rebuy: 5        │
│  ¥150,000                                        Addon: 3        │
│                                                                  │
│   [操作ボタン: スタート / 一時停止 / リセット]                   │
└──────────────────────────────────────────────────────────────────┘
```

**表示要素（領域別）**

| 領域 | 内容 |
|------|------|
| 左上 | 店舗ロゴ画像（最大幅は画面幅の約10%） |
| 中央上 | イベント名（大）、サブタイトル、現在レベル番号（サブタイトル横、小さめ） |
| 右上 | NEXT BREAK IN: 次のブレイクまでの残り時間（HH:MM:SS、ブラインド構造から自動計算） |
| 左中央〜下 | 賞金構造を順位順に縦並び（1位／2位／3位…） |
| 中央 | 大型タイマー（画面高の40〜50%）。残り10秒で赤、ブレイク中はゴールド色 |
| 中央下 | BLINDS カードと NEXT LEVEL カードを**縦2段**で配置。各カード内は「ラベル → SB/BB → ANTE」の縦並び（ANTE は主値の下）。ANTE が 0 の場合は ANTE 行ごと非表示。NEXT LEVEL カードは BLINDS の約 85% に視覚縮小（transform: scale）|
| 右中央 | AVG STACK（平均スタック、数値大きめ） |
| 右下 | PLAYERS（残り/総数）、Rebuy 件数、Addon 件数 |
| 左下 | TOTAL PRIZE POOL（賞金プール総額、大きめ） |
| 画面右下フッタ | 操作ボタン（スタート / 一時停止 / リセット）。マーキー直上、ショートカットガイドの右隣に小さく配置（背景透明・ゴールド極細ボーダー、スタートのみアクティブ時にゴールド充填） |
| 画面下部中央〜左 | 操作ヒント（ショートカットガイド、小） |
| ブレイク中 | タイマー上部に「ブレイク中」ラベル表示、タイマー数字はゴールド色 |
| 一時停止中 | タイマー領域に半透明オーバーレイ + 「一時停止中」表示 |
| 画面最下段 | マーキー（テロップ）領域、`position: fixed`、高さ 5vh、半透明濃紺＋上端ゴールド線。空文字または OFF 時は領域ごと非表示。操作バーの下に重ねる |

**フラッシュ警告**
- 残り 60 秒以下: タイマー文字を黄色に
- 残り 10 秒以下: 画面全体が赤く点滅 (0.5 秒周期)

### 4.2 設定画面 (管理画面)
タブ切替型で以下を管理する。画面右下に「PokerTimerPLUS+ について」リンクを配置（§15.3 参照）。

#### 4.2.1 トーナメントタブ
- 現在のトーナメント情報入力フォーム
  - イベント名 / サブタイトル
  - 通貨記号
  - バイイン金額 / チップ数
  - リバイ金額 / チップ数
  - アドオン金額 / チップ数
- 「クロック開始」「リセット」ボタン

#### 4.2.2 ブラインド構造タブ
- 構造プリセット一覧 (左側)
- 選択したプリセットのレベル一覧 (右側、表形式)
  - レベル番号 / SB / BB / Ante / 時間(分) / ブレイクフラグ
- 行追加・削除・編集
- 新規プリセット作成 / 複製 / 削除
- JSON ファイルとして書き出し / 読み込み

#### 4.2.3 ペイアウトタブ
- ペイアウトプリセット一覧
- 順位ごとの % または固定額編集
- 丸め単位設定 (1 / 100 / 1,000)
- 残り人数別の自動切替ルール (例: 9人以下なら 3位まで、10人以上なら 5位まで)

#### 4.2.4 表示設定タブ
- ロゴ画像選択（任意の店舗ロゴをアップロード、初期値は同梱PLUS2ロゴ）
- 背景プリセット選択（8種、CSSのみで実装、画像不要、べた塗は black のみ、他はグラデーションまたはパターン）
  - `black`: ピュアブラック（唯一のソリッド）
  - `navy`: ネイビー（放射グラデーション、デフォルト）
  - `carbon`: カーボン（チャコール + grid 線）
  - `felt`: フェルトグリーン（深緑 + 斜め織り目）
  - `burgundy`: バーガンディ（斜めグラデーション）
  - `midnight`: ミッドナイト（紫→紺→黒の放射グラデーション）
  - `emerald`: エメラルド（縦グラデーション）
  - `obsidian`: オブシディアン（黒 + ヴィネット）
  - サムネイル 4 列 × 2 段（実背景CSSを縮小再現）クリックで即時反映、選択中はゴールドボーダー
  - electron-store の `display.background` に保存（default `"navy"`）
- フォントサイズ倍率
- 表示要素の ON/OFF (ペイアウト表示など)

#### 4.2.5 マーキータブ
- ON/OFF 切替（チェックボックス）
- テキスト入力（textarea、複数行可、改行は半角スペース3つで連結して1行表示）
- スクロール速度選択（ラジオボタン: 遅い 30秒/周 / 普通 20秒/周 / 速い 12秒/周）
- 「プレビュー」ボタン: 現在のフォーム値をメインクロック画面に即時反映＆永続化
- ショートカット `Ctrl+T` で同じ編集ダイアログを直接開ける（運用中の即時編集）

#### 4.2.6 音設定タブ
- レベル終了チャイム選択 / 試聴
- 1分前警告音 ON/OFF
- 10秒前警告音 ON/OFF
- 5秒前カウントダウン音 ON/OFF
- スタートカウントダウン音 ON/OFF
- マスター音量

### 4.3 サブ画面
- **スタートカウントダウン画面**: トーナメント開始時に "3 → 2 → 1 → スタート!" を画面いっぱいに表示。音と同期。
- **確認ダイアログ**: リセット時、設定変更時の警告。
- **キーガイドオーバーレイ**: F1 キーで表示／非表示。ショートカット一覧を半透明で重ねる。
- **About画面**: 設定画面の「PokerTimerPLUS+ について」リンクから表示（§15.2 参照）。

```json
{
  "logo": {

    "countdown5SecEnabled": true,
```

- `logo.useDefault`: `true` のとき同梱PLUS2ロゴを初期表示する。ユーザーがアップロード後は `false` になる。

## 7. ショートカットキー一覧

PokerStars Poker Clock のキー体系を踏襲しつつ、日本語キーボードでも誤爆しないよう調整する。

| キー | 動作 |
|------|------|
| Space | 一時停止 / 再開 |
| Enter | スタート (待機状態のとき) |
| F1 | キーガイド表示 |
| F2 | メインクロック ⇄ 設定画面 切替 |
| Ctrl+T | マーキー（テロップ）編集ダイアログを直接開く |
| S | 設定ダイアログ（表示タブ）を開く |
| F11 | フルスクリーン切替 |
| Ctrl+Q (macOS: Cmd+Q) | アプリ終了 (確認あり) |
| F12 | DevTools 開閉 (開発モードのみ、本番ビルドでは無効) |
| ← | 残り時間 -1 分（一時停止中も動作） |
| → | 残り時間 +1 分（一時停止中も動作） |
| ↑ | 残り人数 +1 |
| ↓ | 残り人数 -1 |
| Ctrl+R | リバイ +1 |
| Ctrl+Shift+R | リバイ -1 |
| Ctrl+A | アドオン +1 |
| Ctrl+Shift+A | アドオン -1 |
| Ctrl+H | ペイアウト表示 ON/OFF |
| Ctrl+M | 音 ON/OFF (ミュート) |
| Ctrl+Z | 直前の操作を元に戻す (Phase 2 推定) |
| Esc | ダイアログを閉じる / オーバーレイを閉じる |

**実装ポリシー**
- Web Audio API ベースで実装。`AudioContext` は最初のユーザー操作後に初期化。
- 音源は `src/audio/` にバンドル。差し替えは設定画面から ローカルファイル参照。
- 同時再生時は最後の音を優先 (前の音はフェードアウト)。
- 5秒カウントは `count-tick.mp3` を 5/4/3/2/1 秒のタイミングで都度トリガする方式（個別ファイルではなく単一ファイルの繰り返し再生）。

## 16. Phase 2 候補（日本ポーカークラブ運用調査より）

STEP 6.7 で実施した日本のアミューズメントポーカークラブ運用調査の結果を踏まえ、Phase 2 で検討する機能を以下に記録する。

### 優先度高

- **営業時間自動カウントダウン**: 深夜0時終了などへの自動通知。閉店時刻を設定すると、現在のレベル進行とは別に「閉店まで残り時間」を表示し、一定時刻でアラート。
- **バウンティトーナメント管理モード**: バウンティ額・対象プレイヤー名の表示、脱落時の自動通知、累積バウンティ額の集計。
- **シフト交代ロック機能**: PIN保護で誤操作防止。スタッフ交代時に設定変更や時間操作を一時的にロックし、暗証番号入力で解除。

### 優先度中

- **複数卓同期進行**: ハンド・フォー・ハンド対応。複数ディスプレイ間でレベル進行を同期し、フィニッシュテーブル前後の挙動制御。
- **リーグ戦ポイント計算**: 順位別ポイントの自動付与・累積管理。月間／シリーズ単位の累積表エクスポート。
- **サテライト・本戦チェーン**: サテライト優勝者を本戦に自動エントリー。エントリー権の発行・管理。

### 優先度低

- **最終テーブル表示**: ファイナルテーブルのハイライト演出（プレイヤー数が一定以下になったら自動切替）。
- **フリーロール・トーナメント区別フラグ**: 賞金区分とは別軸の運用属性タグ。

> 上記の機能は STEP 6.7 時点では未実装。実装時は本仕様書に詳細仕様を追記する。
