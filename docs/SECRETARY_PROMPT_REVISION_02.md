## タスク 1: 新規スライド 5「2 画面で運用する（HDMI 接続）」

### 配置位置
既存スライド 4 と 5 の間に挿入。既存 slide_05〜slide_11 のオブジェクト ID は新スライド追加に伴って 1 ずつシフト（slide_06〜slide_12）。

### 画像素材
保存先: `poker-clock/docs/slides_artifacts/v206-screenshots/`
- `01-display-extend.png`: Windows の表示設定で「拡張」が選ばれている画像
- `02-monitor-select.png`: アプリ起動時の「ホール側のモニターを選択」ダイアログ
- `03-operator-pane.png`: 手元 PC の operator-pane（運用情報 + 操作一覧）

### スライド構成（3 ステップ + 3 画像）

**スライドタイトル**: `2 画面で運用する（HDMI 接続）`

**リード文**（タイトル直下）:
```
HDMI ケーブルで会場モニターを繋ぐと、自動的に「会場側のタイマー画面」と「手元 PC の操作画面」を分けて表示できます。
```

**3 ステップ**:

| ステップ | 見出し | 説明文 | 画像 |
|---|---|---|---|
| 1 | **準備**: PC を「拡張」表示に設定 | HDMI ケーブルで会場モニターを PC に接続 → **Windows キー + P** を押して表示モードを「**拡張**」に切り替え（複製や PC 画面のみではなく拡張） | `01-display-extend.png`（右側 or 下部に配置）|
| 2 | **アプリ起動**: モニターを選ぶ | アプリを起動すると「**ホール側のモニターを選択**」ダイアログが出る → お客様に見せる側のモニターを選択ボタンで指定。「キャンセル」を押すと従来の単画面モードで起動 | `02-monitor-select.png` |
| 3 | **手元 PC で操作** | 会場モニター側はタイマーが**全画面表示**（単画面モードと同じ表示）、手元 PC 側には**運用情報 + 操作一覧**が表示される。HDMI を抜くと自動的に単画面モードに切り替わる（タイマーは途切れず継続） | `03-operator-pane.png`（手元 PC 画面）|

### レイアウト指示
- 既存スライドの Premium ダーク + Gold アクセントを踏襲（背景色・フォント・縦アクセントバー）
- 1 ステップ = 1 行（縦方向に 3 行配置）or 1 ステップ = 1 カラム（横 3 カラム配置）、CC が 3 画像のサイズと収まりを見て判断
- 画像は縦横比を保ったまま、視認性を保つサイズで配置（小さすぎないこと）
- ステップ番号（1 / 2 / 3）は Gold 色の丸数字 or 番号バッジで強調

### 重要な補足（説明文に含める or 注記）
- HDMI ケーブルが繋がっていない PC では、このダイアログは出ない（自動的に単画面モードで起動）
- HDMI を抜き差しすると自動的にレイアウトが切り替わる（タイマーは途切れない、v2.0.4 以降の機能）

## 実装フロー

1. **アーティファクトの `slide_NN.json` 確認**:
   - `slide_01.json`（タイトル）/ `slide_04.json`（画面の見方）/ `slide_11.json`（クレジット、現状）を Read
   - `object_id_map.md` で各スライドのオブジェクト ID を確認
2. **新規 `slide_05.json` 作成**:
   - 既存スライド（例: slide_03 や slide_07 等のシンプル構成スライド）の JSON 構造を参考に、3 ステップ構成で新規作成
   - 画像 3 枚を `image_urls.json` に登録（必要なら Drive アップロード経由）
3. **既存スライド 5〜11 の繰り下げ**:
   - 既存ファイルをリネーム（slide_05.json → slide_06.json、…、slide_11.json → slide_12.json）
   - `build_slides.py` のスライド数定義 / object_id_map を併せて更新
4. **タイトル / クレジット / 画面の見方の修正**:
   - slide_01.json / slide_04.json / slide_12.json（旧 slide_11）に必要な変更を加える
5. **再構築**:
   - `python build_slides.py --slides 1,4,5,6,7,8,9,10,11,12 --skip-upload --clear`（全スライド or 影響範囲のみ）
   - もしくは全スライドを再構築（全 12 枚）
6. **サムネイル更新**:
   - 影響を受けたスライドのサムネイル（少なくとも slide_01 / slide_04 / slide_05 / slide_12）を再生成
7. **完了報告**:
