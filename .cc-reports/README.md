# .cc-reports/

CC (Claude Code) の作業完了報告を書き出すフォルダ。

## 運用ルール
- ファイル名: `YYYY-MM-DD_案件ID_短い作業名.md`
- 書き手: CC (作業完了時に書き出し)
- 読み手: CC構築士2 (前原から「CC作業完了」と一言聞いたら最新を Read)
- 内容: やったこと / 検証結果 / 構築士2 への確認事項 / 残作業 / 触ったファイル一覧
- このフォルダの中身は `.gitignore` 対象。`README.md` と `.gitkeep` のみコミット対象。

## 報告フォーマット
```
# CC作業完了報告 — YYYY-MM-DD 案件ID 短いタイトル

## 1. サマリー
(1〜2行で何をしたか)

## 2. 触ったファイル
| ファイル | 変更点 |

## 3. 検証結果
(テスト・動作確認の結果)

## 4. 構築士2 への確認事項
(あれば。なければセクション省略)

## 5. 残作業 / 次にできそうなこと
(スコープ外で気づいたことは提案として記載のみ、勝手に実装しない)
```
