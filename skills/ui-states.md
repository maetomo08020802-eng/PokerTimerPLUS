# UI状態とアニメーション

## タイマー警告色

| 残り時間 | 色 | 効果 |
|---|---|---|
| > 60秒 | `--text-primary` 白 | グロー（金）控えめ |
| 10〜60秒 | `--warn` #FFC107 | グロー（黄）強め |
| 0〜10秒 | `--danger` #E53935 | 0.5秒周期の点滅＋拡大1.05倍 |

## タイマーグローのクリッピング防止（必達）
グロー（text-shadow）はテキストボックスを超えて拡散するため、
`.clock__timer` および その親までのチェーンに `overflow: hidden` がかかっていると
グローの端がクリップされる。以下を必ず守る：

- `.clock__timer` には `overflow: visible` を明示
- `.clock__center`（タイマーを内包するカラム）も `overflow: visible` にする、
  または padding で 8vw 程度の余白を確保してグローを内側に収める
- 全画面（F11）でも vw 単位の glow が比例拡大しても切れないこと
- レイアウト安定性を維持しつつ glow をクリップしないため、
  「中央列だけ overflow: visible、左右列は overflow: hidden」の選択的適用が望ましい

## 一時停止中
- 全体に半透明オーバーレイ `rgba(0,0,0,0.5)` を `position: absolute; inset: 0; pointer-events: none;` で重ねる
- 既存レイアウトに影響しないこと
- タイマー文字を `filter: grayscale(40%)`（transform は使わない、sizeも変えない）
- 「一時停止中」のラベルはオーバーレイ内に絶対配置
- **既存のflex/grid要素として挿入してはならない**
- 一時停止のON/OFFで他要素のサイズ・位置がズレないこと（layout shift禁止）

## ブレイク中
- タイマー本体は通常通り MM:SS 表示を続ける（数字のサイズ・位置は変えない）
- 「ブレイク中」ラベルは**absolute 配置のオーバーレイ**として、タイマーの上または下に重ねる
  - `position: absolute; top: -3vh;` 等で配置
  - **document flow に影響させない**
  - 既存のflex/grid子要素として挿入してはならない
- 背景色をやや明るくしたい場合は body 全体の background-color を `--bg-elevated` に変える（要素の追加はしない）
- BLINDSカードを「ブレイク」と表示する場合、カード内テキストの差し替えで対応（カードの追加・削除はしない）
- ブレイクのON/OFFで他要素のサイズ・位置がズレないこと

## アニメーション

### OK（採用）
- タイマー警告色のフェード遷移（0.3秒、ease-out）
- カード hover時の背景色トランジション（0.2秒）
- 残り10秒以下の点滅（0.5秒周期）
- ブレイク開始時のフェードイン（0.4秒）

### NG（禁止）
- タイマー数字の遅延・スライド表示（1秒は1秒で動く）
- 過剰なパーティクル・グラデーション動的変化
- 起動時の派手なロゴアニメーション

## 禁止事項（アニメーション・状態）
- アニメーションで残り時間表示を遅延させない（1秒は1秒で動く）
- モーダル多用しない（操作のテンポを止める）
- 警告色を多用しない（赤は最後の10秒のみ）
- 影・グローを過剰に使わない（テキストシャドウは0.3 alpha以下）
- 背景画像は控えめに（読みやすさ優先、暗いオーバーレイ必須）
- カード背景の透明度を下げすぎない（最低 0.55 alpha）

## アクセシビリティ
- ボタンは最小48x48px（CSS: `min-width: 3vw; min-height: 3vw`）
- 文字色とのコントラスト比4.5:1以上（白×濃紺で達成）
- キーボードのみで全操作可能
- カード内テキストは backdrop blur がOFFの環境でも読める透明度に
