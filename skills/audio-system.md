# 通知音システム品質基準

## 通知音の鳴動タイミング
1. レベル終了10秒前: 通知音A（短い「ポーン」）1回
2. レベル終了5秒前から1秒前: 通知音B（カウント音）毎秒1回（計5回）
3. レベル変更瞬間: 通知音C（強めの「ジャーン」）1回
4. スタートカウントダウン3-2-1: 通知音B 3回 + レベル開始時に通知音C
5. ブレイク開始: 通知音D（柔らかい「チャイム」）1回
6. ブレイク終了10秒前: 通知音Aを流用

## 実装ルール
- Web Audio API（AudioContext）を使用、HTMLAudioElementは避ける
- 音源は事前にdecodeAudioDataでメモリにロード（再生遅延ゼロ）
- 同時多重再生に耐える設計（前の音が鳴っていても被せて再生）

## ミュート機能
- マスターミュート（全体）
- 個別ミュート（カウント音のみ等）
- 設定はelectron-storeに永続化

## 音源仕様
- フォーマット: mp3（44.1kHz / 128kbps以上）
- 長さ: 通知音Aは0.5秒、Bは0.3秒、Cは1.0秒、Dは1.5秒以内
- 音量: 全音源を-3dBにノーマライズ
- 配置: src/audio/ 配下
- ファイル名: notify-warn.mp3 / notify-count.mp3 / notify-change.mp3 / notify-break.mp3

## 禁止事項
- ユーザー操作なしの自動再生はしない（ブラウザポリシー対策で初回操作時にAudioContext.resume）
- 音源を外部URLから取得しない（オフライン動作必須）
- ボリューム0%設定時にAudioContextを動かさない（CPU節約）
